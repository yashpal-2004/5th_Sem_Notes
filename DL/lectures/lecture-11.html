Limitations of ANNs for Vision

1.1 Loss of Spatial Relationships

Standard Artificial Neural Networks (ANNs) process images by flattening the 2D or 3D pixel grid into a 1D vector. This has severe limitations for vision tasks:

  • Disregard of Neighborhood: Flattening treats pixels that were adjacent vertically or diagonally in the 2D grid as completely independent features, failing to exploit the spatial correlations inherent in images.
  • Lack of Translation Invariance: If a visual pattern (like a cat) shifts by a few pixels, the flattened vector changes completely. A standard ANN has to relearn the pattern for every possible location in the image.

1.2 Parameter Explosion

In a fully connected layer, every input pixel connects to every hidden neuron. For a modest color image of size 256 × 256 × 3, the input size is 196,608. Connecting this to a hidden layer of 1,000 units requires over 196 million weights for a single layer, leading to massive memory costs and high overfitting risks.

Biological Motivation

2.1 Hubel & Wiesel Experiments

In the 1950s/60s, neuroscientists Hubel and Wiesel studied the visual cortex of cats and made key discoveries about biological vision:

  • Local Receptive Fields: Neurons do not process the entire visual scene simultaneously. Instead, each neuron responds only to a tiny, localized region of the visual input (receptive field).
  • Orientation Selectivity: Neurons fire selectively in response to edges and lines oriented at specific angles, rather than individual raw pixels.

2.2 Simple vs. Complex Cells

The visual cortex utilizes two primary classes of cells to build translational invariance:

  • Simple Cells: Fire only when they observe a line/edge at a specific angle in a specific, fixed spot within the receptive field.
  • Complex Cells: Have larger receptive fields. They fire when they observe a line/edge at a specific angle anywhere in their receptive field, establishing translation invariance.
  • Hierarchical System: Subsequent layers combine outputs from complex cells to detect more complicated shapes (curves, textures) and eventually entire objects.
3

Invariance vs. Equivariance

3.1 Spatial Properties in Image Tasks

We describe the relationship between input translations and output targets using two mathematical properties:

Definitions
  • Invariance: A function f(x) is invariant to translation g if applying translation to the input does not alter the output:
    f(g(x)) = f(x)
    Useful for classification (e.g. classifying "cat" regardless of where the cat is located in the image).
  • Equivariance: A function f(x) is equivariant to translation g if translating the input translates the output in the exact same way:
    f(g(x)) = g(f(x))
    Useful for semantic segmentation or object detection (where we expect boundary overlays to shift as the image shifts).

Convolution Mechanics

4.1 Kernel Size, Stride, and Dilation

Convolutional layers slide a weight matrix (filter or kernel) over the input. They compress fully connected layers by using sparse connectivity (each hidden unit only looks at a small input patch) and weight sharing (the same filter weights are reused across the entire image). The knobs that control this sliding operation include:

  • Kernel Size (F): Spatial dimensions of the filter patch (e.g. 3 × 3 or 5 × 5).
  • Stride (S): Number of pixels the filter shifts between updates (e.g. stride S=1 slides continuously; stride S=2 skips every other pixel, downsampling the output).
  • Dilation (D): Spacing between weights in the filter. Dilation D=1 is normal convolution. Dilation D=2 inserts a zero weight between each kernel weight, expanding the receptive area of the filter without adding learnable parameters.

4.2 Grayscale vs. RGB Convolutions

  • Grayscale Images: Inputs are 2D grids (height × width). Filters are 2D (e.g. 3 × 3).
  • Color Images: Inputs are 3D volumes (height × width × channels (RGB)). The filter must match the input depth, having dimensions height × width × 3. The 3D filter is basically three stacked 2D filters that calculate a single combined output value for each receptive field.

4.3 Receptive Field Growth

The receptive field (RF) is the size of the region in the original input that contributes to the calculation of a hidden unit activation. In fully connected layers, the receptive field is always 100% of the input. In CNNs, the receptive field grows larger in deeper layers:

  • Layer 1: (Kernel size 3) → Receptive field = 3.
  • Layer 2: (Kernel size 3, Stride 1) → Receptive field = 5.
  • Layer 3: (Kernel size 3, Stride 2) → Receptive field = 7.
  • As layers stack, the high-level units eventually cover the entire image, allowing the network to synthesize global context.
5

Padding & Pooling

5.1 Zero Padding and Boundary Checks

Performing convolutions repeatedly causes the spatial dimensions of the outputs to shrink. If the input is N × N and the filter is F × F, the output size is (N - F + 1) × (N - F + 1). To address this, we use zero padding (P) to add zero pixels around the image borders:

Output Width = [ (N - F + 2P) / S ] + 1
Why Use Padding?
  • Size Preservation: Selecting padding P = (F - 1) / 2 for filter size F and stride S = 1 keeps the output spatial dimensions identical to the input.
  • Edge Preservation: Without padding, the corners of the image are only visited once by the corner of the filter, while central pixels are calculated repeatedly. The network becomes "blind" to border information. Padding lets the filter slide past borders, evaluating edges equally.

5.2 Max Pooling & Subsampling

Max Pooling is a non-learnable downsampling operation that takes a local patch (e.g. 2 × 2) of a feature map and outputs only the maximum activation value:

  • Dimensionality Reduction: Reduces the height and width of feature maps, lowering parameter count and computing cost in subsequent layers.
  • Translation Invariance: Because max pooling only cares about the maximum value in the patch, a small shift in the input activations does not change the pool output.
