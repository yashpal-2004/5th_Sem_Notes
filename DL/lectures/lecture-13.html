U-Net Architecture

1.1 Symmetrical Encoder-Decoder

ResNets are highly effective at classification but lose the spatial resolution required for pixel-perfect localization. To solve this, Ronneberger et al. (2015) introduced U-Net for semantic segmentation (classifying every individual pixel in an image):

  • U-Shaped Symmetry: Symmetrical structure consisting of a contracting path (encoder) and an expanding path (decoder).
  • Encoder (Down-sampling): Extracts high-level semantic features (the what) using standard convolutions and pooling, reducing spatial size.
  • Decoder (Up-sampling): Restores spatial resolution (the where) back to the original image dimensions using transposed convolutions.

1.2 Skip Connections for Localization

As features pass through the encoder's bottlenecks, high-resolution spatial information is lost. U-Net solves this by introducing horizontal skip connections that concatenate the high-resolution feature maps from the encoder directly to the decoder's up-sampled feature maps. This allows the decoder to retain fine-grained spatial details, making U-Net highly successful for biomedical imaging segmentation.

Inception Network (GoogleNet)

2.1 Parallel Filters & Bottlenecks

Introduced by Google in 2014, the Inception network shifted the design focus from stacking layers deeper to building wider networks. The core unit is the Inception Module:

  • Parallel Processing: Instead of choosing a single filter size, it applies 1 × 1, 3 × 3, and 5 × 5 convolutions, and 3 × 3 max pooling in parallel on the same input, capturing features at multiple scales simultaneously.
  • 1 × 1 Bottleneck Convolutions: Applying 3 × 3 or 5 × 5 filters on thick input volumes is computationally expensive. Inception uses 1 × 1 convolutions to reduce channel depth before applying larger filters, drastically lowering computational cost.

2.2 Inception Shortcomings

Despite strong feature extraction, Inception models face limitations in object detection:

  • Context Blindness: By processing local windows at multiple scales, Inception struggles with global scene context, sometimes mistaking background patches for objects.
  • Complexity & Latency: The multiple branching paths within each module create operational overhead, making high frame-rate real-time deployment difficult.

R-CNN & Fast R-CNN

3.1 Region Proposals & Selective Search

Early object detection models used slow "sliding window" sweeps. The R-CNN family introduced regional shortcuts:

  • R-CNN (Region-based CNN): Uses a fast external algorithm (Selective Search) to identify approximately 2,000 candidate bounding boxes (Regions of Interest, RoI) likely to contain objects. It crops these regions out and passes each crop separately through a CNN to classify the object and adjust the bounding box coordinates.
  • Limitation: Running 2,000 separate CNN forward passes per image is extremely slow, taking up to 47 seconds per image.

3.2 RoI Pooling Mechanics

Fast R-CNN resolved the speed bottleneck by passing the entire image through the CNN backbone just once to extract a global feature map. It maps the candidate bounding boxes onto this feature map. Region of Interest (RoI) Pooling then extracts a fixed-size (e.g. 7 × 7) feature vector from each mapped region, allowing classification and bounding-box regression heads to process all regions simultaneously. This speed improvement makes training and inference much faster.

4

Faster R-CNN Evolution

4.1 Region Proposal Networks

While Fast R-CNN made feature extraction faster, it still relied on the slow Selective Search algorithm outside the network. Faster R-CNN eliminated this bottleneck by integrating a Region Proposal Network (RPN) directly into the CNN architecture:

  • Anchor Boxes: The RPN slides a small network over the global feature map. At each position, it evaluates multiple predefined bounding boxes of different scales and aspect ratios (anchors) to score object presence and suggest bounding box offsets.
  • End-to-End Training: Because the region generation and classification heads share the same feature map backbone, the entire system is trained end-to-end, enabling near real-time performance.
5

YOLO & SAM Models

5.1 YOLO: Single-Stage Detection

Faster R-CNN is a two-stage detector (first proposing regions, then classifying them). YOLO (You Only Look Once) framing object detection as a single regression problem:

  • Grid-based Evaluation: YOLO divides the input image into an S × S grid. If an object's center falls into a grid cell, that cell is responsible for predicting the bounding boxes and class probabilities directly in a single forward pass.
  • Speed Advantage: By bypassing the region proposal stage entirely, YOLO achieves extremely high frame rates (e.g. 45+ FPS), making it ideal for real-time video applications.

5.2 Segment Anything Model (SAM)

Developed by Meta in 2023, the Segment Anything Model (SAM) represents a breakthrough in foundation models for computer vision segmentation:

  • Promptable Segmentation: Users can guide segmentation by providing input prompts such as points, bounding boxes, or text queries.
  • Zero-Shot Generalization: SAM is trained on massive datasets (11 million images, 1 billion masks) and can segment novel objects and environments without fine-tuning.
