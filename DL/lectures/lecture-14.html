Sequence Data & FFNN Failures

1.1 Variable Input Lengths & Padding

Real-world sequences (like text sentences, audio waves, video frames, stock prices, or DNA strands) vary in length. Standard feedforward networks (FFNNs) are poorly suited for sequence data due to two main limitations:

  • Weight Dimension Changes: If sentences have different numbers of words, a standard FFNN's input weight dimensions would have to change for each sentence, which is mathematically impractical.
  • Zero-Padding Inefficiencies: To keep input sizes constant, we can pad shorter sentences with zeroes. However, if the average sentence length is 50 words but the longest sentence is 1,000, padding creates massive, sparse matrices and leads to wasted computational steps on zero values. Furthermore, the model cannot handle predictions longer than its fixed maximum padded size.

1.2 Ignoring Relative Word Order

A standard FFNN processes inputs simultaneously and does not capture positional order. If we shuffle the words in a sentence, a fully connected network produces the exact same activation profile, failing to understand how word order alters grammatical meaning (e.g. "dog bites man" vs. "man bites dog").

Recurrent Neural Networks (RNN)

2.1 Recurrent Cells & Hidden States

To process sequential data, we need networks that utilize context from previous steps. Recurrent Neural Networks (RNNs) achieve this by introducing a feedback loop within the hidden units:

  • Hidden State (ht): Acts as the network's memory. At each time step t, the recurrent cell computes a new hidden state based on the current input vector xt and the previous hidden state ht-1:
    ht = tanh(Whh ht-1 + Wxh xt + bh)
  • The hyperbolic tangent (tanh) activation function maps outputs to a [-1, 1] range, regulating values over time steps.

2.2 The Principle of Parameter Sharing

Instead of learning different weight matrices for each word position in a sentence, an RNN uses the exact same weight matrices (Whh, Wxh, and output weights Why) across all time steps. This parameter sharing has two key benefits:

  1. Generalization: It allows the network to process sequences of any length without needing additional parameters.
  2. Size Reduction: It drastically reduces the model parameter count compared to fully connected architectures.
3

RNN Output Topologies

3.1 Sequence Mapping Types

RNNs are highly flexible and can configure input-to-output mappings for different tasks:

  • One-to-One: Standard feedforward processing (e.g. classification on static vectors).
  • One-to-Many: A single input maps to a sequence of outputs (e.g. image captioning).
  • Many-to-One: A sequence of inputs maps to a single output (e.g. sentiment classification on a review).
  • Many-to-Many:
    • Synced (same length): Inputs and outputs map step-by-step (e.g. POS tagging).
    • Unsynced (different lengths): The entire input sequence is processed before generating the output sequence (e.g. machine translation via encoder-decoders).

Parameter Counting Case Study

4.1 1D Review Model Parameter Derivation

Let's calculate the total number of learnable parameters in a simple RNN built for sentiment classification:

Model Specifications
  • Vocabulary Size (Input features): 5 unique words, one-hot encoded: size = 5.
  • Hidden Layer Dimension: 3 neurons (hidden state vector ht size = 3).
  • Output Layer Dimension: 1 neuron (sigmoid probability output yt size = 1).

To compute the total learnable parameters, we sum the weights and biases:

  1. Input to Hidden Weights (Wxh): Size = (hidden_dim × input_dim) = 3 × 5 = 15 weights.
  2. Hidden Recurrent Weights (Whh): Size = (hidden_dim × hidden_dim) = 3 × 3 = 9 weights.
  3. Hidden Biases (bh): Size = hidden_dim = 3 biases.
  4. Hidden to Output Weights (Why): Size = (output_dim × hidden_dim) = 1 × 3 = 3 weights.
  5. Output Bias (by): Size = output_dim = 1 bias.
  6. Total Parameters: 15 + 9 + 3 + 3 + 1 = 31 parameters.

BPTT & Gradient Problems

5.1 Backpropagation Through Time

To train an RNN, we unfold the computational graph across all time steps of a sequence. The total loss J is the sum of losses at each step (or the final step). Backpropagation is performed from the end of the sequence back to the beginning, a process known as Backpropagation Through Time (BPTT).

5.2 Vanishing & Exploding Gradients

BPTT faces severe numerical issues when processing long sequences. Due to parameter sharing, the recurrent weight Whh is applied repeatedly. Applying the chain rule to propagate the loss gradient from step T back to step t yields a product of jacobian matrices:

∂hT / ∂ht = ∏k=t+1T WhhT ⊙ diag(1 - tanh2(...))

This product leads to two main optimization challenges:

  • Vanishing Gradients: If the largest eigenvalue of Whh is less than 1, multiplying it repeatedly causes the gradients to shrink exponentially to zero. The network cannot propagate error information back to early steps, making it unable to learn long-term dependencies (only short-term dependencies).
  • Exploding Gradients: If the largest eigenvalue of Whh is greater than 1, the product grows exponentially, causing gradients to blow up to infinity and triggering NaN values or optimization instability.
