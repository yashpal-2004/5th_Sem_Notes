Seq2Seq Architecture

1.1 Encoder-Decoder Separation

Standard RNNs require a 1-to-1 mapping between input and output tokens, making them unable to handle tasks like translation where sentences have different lengths and grammar ordering (e.g. "I love India" translates to "Mujhe Bharat pasand hai"). To solve this, Sutskever et al. (2014) introduced the Sequence-to-Sequence (Seq2Seq) architecture:

  • Encoder: An RNN that reads the entire input sequence step-by-step, compressing it into a final hidden state vector (context vector) containing global semantic context.
  • Decoder: A separate RNN that takes the context vector as its initial state and generates the output sequence one token at a time.

1.2 Special Alignment Tokens

Seq2Seq models use special tokens to coordinate translation boundaries:

  • START: A special token fed to the decoder's first step to trigger the generation process.
  • EOS (End of Sequence) / END: Appended to the end of sentences during training. When the decoder predicts this token during inference, it stops generating.
2

Teacher Forcing Mechanics

2.1 Training ground truth feeding

During training, the decoder makes predictions step-by-step. If it predicts a wrong word at step t, feeding this incorrect word into step t+1 would cause errors to cascade, making training slow and unstable. To solve this, we use Teacher Forcing: we feed the actual ground truth word from the training set into the decoder at step t+1, regardless of whether its prediction at step t was correct. This ensures faster convergence.

2.2 Prediction autoregressive loops

During the prediction (inference) phase, ground truth outputs are unavailable. The decoder operates in an autoregressive loop: the token predicted at step t is fed as the input to step t+1. Under this setup, early prediction errors propagate forward, sometimes degrading output quality on long sentences.

3

The Information Bottleneck

3.1 Fixed Context Vector limits

A key limitation of standard Seq2Seq models is the information bottleneck problem. The encoder compresses the entire input sequence into a single, fixed-size context vector. As the input sentence length increases, the amount of information lost grows, causing translation performance to drop on sentences longer than ~20 words. Compressing a whole paragraph into one vector is like summarizing a page in a single word—loss of detail is inevitable.

4

Bahdanau Additive Attention

4.1 Feedforward alignment models

To solve the bottleneck, Bahdanau et al. (2014) introduced the Attention Mechanism. Instead of relying on a single context vector, the decoder dynamically accesses all encoder hidden states at each step. Bahdanau (Additive) attention uses a learnable feed-forward neural network to calculate alignment scores between the current decoder state st-1 and each encoder state hi:

score(st-1, hi) = vaT tanh(Wa st-1 + Ua hi)

Where vaT, Wa, and Ua are learnable weight parameters.

4.2 Dynamic context updates

These alignment scores are passed through a softmax function to produce attention weights αti, which sum to 1. The model computes a dynamic context vector ct as the weighted sum of all encoder states:

ct = ∑i αti hi

This allows the decoder to "attend" directly to relevant parts of the input sentence (e.g. focusing on the word "cats" when generating "billi").

5

Luong Multiplicative Attention

5.1 Dot-product scaling scoring

Luong et al. (2015) proposed a simpler, faster alternative called Luong (Multiplicative) Attention. Instead of using a feed-forward network, it calculates alignment scores using dot products or matrix multiplications between the current decoder state st and encoder states hi:

  • Dot Score: The simplest form:
    score(st, hi) = stT hi
  • General Score: Uses a learnable weight matrix W to align different dimensional spaces:
    score(st, hi) = stT W hi

5.2 Comparison with Bahdanau

Feature Bahdanau (Additive) Luong (Multiplicative)
Alignment Score vaT tanh(Wa st-1 + Ua hi) stT hi  or  stT W hi
Decoder State Uses previous state st-1 Uses current state st
Complexity Higher (trains FFNN parameters) Lower (matrix multiplication)
