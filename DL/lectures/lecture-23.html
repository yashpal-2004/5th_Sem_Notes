Flash Attention

1.1 Memory Hierarchy Bottlenecks (HBM vs. SRAM)

Standard attention is memory-bound rather than compute-bound. Calculating and storing the intermediate N × N attention matrix requires frequent read/write operations to High Bandwidth Memory (HBM), which is much slower than the GPU's internal SRAM cache. This memory transfer bottleneck limits training speed.

1.2 Tiling and Block Loading

Flash Attention resolves this bottleneck by loading inputs in small blocks (tiles) from HBM directly into SRAM. Attention is computed block-by-block without writing the intermediate N × N matrix back to HBM, reducing memory read/writes and accelerating the attention step.

1.3 Online Softmax Updates

Because softmax requires a global normalization factor, computing it block-by-block is challenging. Flash Attention uses an online softmax algorithm, dynamically updating scaling factors as blocks are processed, ensuring exact attention calculation with minimal memory overhead.

Supervised Fine-Tuning (SFT)

2.1 Instruct Tuning Protocols

SFT or instruct tuning aligns pretrained base models with task formats. The model is trained on formatted datasets (input prompt, output response) using standard cross-entropy loss, teaching the model to follow instructions and act as an assistant.

2.2 Data Quality vs. Quantity

Recent research highlights that SFT is highly sensitive to instruction quality rather than instruction count. A dataset of 1,000 highly curated, diverse, and clean instructions often yields better alignment than 100,000 noisy, machine-generated prompt pairs.

Low-Rank Adaptation (LoRA)

3.1 Low-Rank Hypothesis

Aghajanyan et al. showed that weight updates during adaptation have a low "intrinsic dimension". Consequently, we do not need to update all parameters of a weight matrix during fine-tuning. Instead, we can represent the update matrix ΔW in a lower-rank subspace.

3.2 Adapter Matrices Decomposition (A and B)

For a pretrained weight matrix W0 ∈ ℝd × k, we freeze its weights and represent the update ΔW using two low-rank matrices B ∈ ℝd × r and A ∈ ℝr × k, where rank r << min(d, k):

W = W0 + ΔW = W0 + B A

Matrix A is initialized with random Gaussian values, and B is initialized to zero, ensuring that ΔW is zero at the start of training.

3.3 Parameter and Memory Savings

Using a rank r=8 reduces the number of trained parameters by over 99%, lowering optimizer memory requirements and saving significant compute resources during fine-tuning.

4

Knowledge Distillation

4.1 Teacher-Student Compression

Knowledge Distillation is a model compression method where a small model (student) is trained to replicate the behavior and outputs of a larger, pretrained model (teacher).

4.2 Soft Targets and Entropy Transfer

Instead of training the student only on hard labels (ones and zeros), we train it using the soft targets (probability distributions) produced by the teacher's softmax. These soft targets contain valuable dark knowledge (e.g. telling the student that a picture of a dog is 10% similar to a cat, but 0.001% similar to a car), transferring the teacher's entropy and behavior to the smaller model.
