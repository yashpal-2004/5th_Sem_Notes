Space Folding & Composition

1.1 Geometric Composition

Stacking layers in deep networks can be interpreted geometrically as a sequence of function compositions. Each layer transforms the coordinate space, modifying the data before passing it to subsequent layers.

1.2 Folding the Input Space

An intuitive way to visualize deep networks is through the concept of space folding:

  • The first hidden layer "folds" the input space x back onto itself along activation boundaries.
  • This folding maps multiple distinct coordinates in the input space to the same post-activation location.
  • Subsequent layers perform additional folds, allowing a simple linear output function to be replicated across all folded regions.
Space Folding Advantages

Folding allows deep neural networks to represent complex, highly oscillatory, or symmetric patterns with a tiny fraction of the parameters that a flat, unfolded boundary would require.

Depth vs. Width Capacity

The expressiveness and capacity of a neural network are determined by its dimensions: width and depth.

2.1 Role of Width

Width (D): The number of neurons in a hidden layer. Width represents memory and capacity:

  • Increasing layer width allows the model to capture fine-grained variations and independent features simultaneously.
  • Wide networks perform well on simpler tasks (e.g., shallow classification or regression with less complex relationships) where a few wide layers can detect all necessary patterns without hierarchical composition.

2.2 Role of Depth

Depth (K): The number of sequential hidden layers. Depth represents abstraction and provides compositional efficiency:

  • A deep network can represent complex functions using only O(N) parameters, whereas a shallow network would require O(2N) parameters to capture the same complexity.
  • Face Analogy: It is much simpler to define a face abstractly as a composition of high-level features (Eyes + Nose + Mouth) than to describe the color values of every pixel coordinate independently.

2.3 Depth/Width Trade-offs

When designing model architectures, we must balance these metrics:

Architecture Trade-off Summary
  • Go Deeper (Modern Trend):
    • Pros: Higher parameter efficiency, captures hierarchical features, matches real-world compositionality.
    • Cons: Harder to train (susceptible to vanishing/exploding gradients).
  • Go Wider:
    • Pros: Easier to optimize (produces a smoother loss landscape), highly parallelizable on GPUs.
    • Cons: High risk of overfitting, low parameter efficiency for hierarchical patterns.

Predicting Probability Distributions

3.1 Shift from Single Values

A major perspective shift in deep learning is treating the network as a system that predicts probability distributions rather than isolated raw values. Instead of predicting a label directly, the network estimates the parameters of a conditional probability distribution, P(y | x; θ).

3.2 Distribution Modeling

By mapping network outputs to parameters of standard distributions, we unify regression and classification:

  • Binary Classification: We assume the targets follow a Bernoulli distribution. The network predicts the parameter φ (probability of class 1) using a Sigmoid output activation.
  • Regression: We assume the targets follow a Gaussian (Normal) distribution. The network predicts the mean μ of the distribution using a Linear output activation.
4

MLE & Negative Log-Likelihood

4.1 Likelihood Maximization

Under Maximum Likelihood Estimation (MLE), we optimize the network parameters θ to maximize the probability (likelihood) of observing the actual training labels given our input data:

L(θ) = ∏ P(yi | xi; θ)

4.2 Floating-Point Underflow

We do not maximize the likelihood function directly for two reasons:

  1. Underflow: Individual probability values are small (between 0 and 1). Multiplying thousands of these probabilities repeatedly yields extremely small numbers that trigger floating-point underflow errors in computer hardware.
  2. Mathematical Convenience: Maximizing products is mathematically complex. By taking the logarithm, we convert the product of probabilities into a summation:
    log L(θ) = ∑ log P(yi | xi; θ)

Because optimization libraries are designed to minimize functions (rather than maximize them), we multiply by -1. This yields the Negative Log-Likelihood (NLL) loss function, which we minimize:

J(θ) = - ∑ log P(yi | xi; θ)
5

Universal Loss Recipe Derivations

By applying Negative Log-Likelihood to different output probability distributions, we can derive standard loss functions:

5.1 Binary Cross-Entropy Derivation

For binary classification where targets y ∈ {0, 1}, we assume a Bernoulli trial with probability φ predicted by the network:

P(y | x; θ) = φy (1 - φ)1-y

Taking the negative logarithm of this probability for a single data point:

-log P(y | x; θ) = -log( φy (1 - φ)1-y )
= - ( y log(φ) + (1 - y) log(1 - φ) )

This is exactly the formula for Binary Cross-Entropy (BCE) loss.

5.2 Mean Squared Error Derivation

For regression, we assume the targets follow a Gaussian distribution centered at the predicted value μ with variance σ2:

P(y | x; θ) = [ 1 / √(2πσ2) ] × exp( - (y - μ)2 / (2σ2) )

Taking the negative logarithm:

-log P(y | x; θ) = -log( 1 / √(2πσ2) ) + (y - μ)2 / (2σ2)

Assuming constant variance σ2, the first term is a constant. Minimizing this negative log-likelihood is equivalent to minimizing the remaining term:

Loss ∝ (y - μ)2

This is the Mean Squared Error (MSE) / L2 loss. This proves that MSE is a direct mathematical consequence of assuming Gaussian noise under Maximum Likelihood Estimation.
