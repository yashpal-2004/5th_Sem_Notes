Masked Self-Attention

1.1 Causal Masking Implementation

To resolve the parallel data leakage problem, the decoder uses Masked Self-Attention. We apply a causal mask to the raw attention scores before the softmax operation. This ensures that the representation for the token at position t can only attend to tokens at positions ≤ t.

1.2 Attention Matrix Modification

Mathematically, we create an upper-triangular masking matrix where future positions are filled with -∞, while allowed positions are 0. Adding this mask to the raw query-key dot products results in:

MaskedAttention(Q, K, V) = softmax( [ Q KT / √(dk) ] + M ) V

When passed to softmax, the -∞ values evaluate to exactly 0 probability, blocking any attention flow from future tokens.

Cross-Attention Mechanics

2.1 Encoder-Decoder Information Flow

Cross-attention acts as the bridge connecting the encoder and the decoder. It allows the decoder to query the rich context-aware vectors produced by the encoder, extracting the relevant source details needed to generate the next target token.

2.2 Q, K, V Role Assignments

Unlike self-attention, the inputs to the cross-attention layer are split:

  • The Query (Q) matrix is projected from the decoder's preceding masked attention layer.
  • The Key (K) and Value (V) matrices are projected from the final output of the encoder stack.
This allows each decoder position to scan across all encoder output positions to retrieve relevant context.

3

Decoder Architecture Stack

3.1 The 3 Sub-layer Pipeline

A standard decoder block consists of three sub-layers wrapped in residual connections and LayerNorm:

  1. Masked Multi-Head Self-Attention: Captures context from previously generated target tokens.
  2. Multi-Head Cross-Attention: Connects the decoder to the encoder's source context.
  3. Position-wise Feed-Forward Network: Processes each token vector independently.

3.2 Generative Linear Head

At the top of the decoder stack, the output vectors are mapped to a vocabulary space using a final Linear layer. A softmax function transforms these values into probability distributions over the entire vocabulary, identifying the most likely next token.

4

Autoregressive Inference

4.1 Step-by-Step Auto-Regression

During inference, the model runs auto-regressively:

  1. Input begins with a start token (e.g. <SOS>).
  2. The model predicts the first word.
  3. This predicted word is appended to the input to predict the second word, repeating the process.

4.2 End-of-Sequence (EOS) Control

The generation loop continues until the model outputs a special end-of-sequence token (e.g. <EOS>) or reaches a predefined maximum token limit, terminating the process.

4.3 Error Cascading Analysis

Because predictions feed back into the input sequence, an incorrect selection at step 2 alters the context for all subsequent steps. This cascading error propagation can lead to drifts or repetitive loops during generation.

5

Applications & Transfer Learning

5.1 NLP & Vision Modalities

Transformer decoders power large language models (like GPT) and text-to-image systems (like DALL-E), treating text sequences and image patches as tokenized generation sequences.

5.2 Task vs. Data Transfer

Transfer learning leverages representations learned on a source task to improve learning on a target task:

  • Task Transfer: Reusing representations when the task changes (e.g. pretraining on language modeling, then adapting to classification).
  • Data Transfer: Reusing representations when the data domain changes but the task remains similar.

5.3 Model Fine-Tuning

Instead of training models from scratch, we pretrain a large model on massive datasets (unsupervised learning) and then fine-tune it by training on a smaller, task-specific dataset, adapting the weights to the target domain.
