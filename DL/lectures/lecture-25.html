Under-Complete Autoencoders

1.1 Bottleneck Dimension Constraints

An under-complete autoencoder has a latent bottleneck dimension h that is smaller than the input dimension x. This bottleneck forces the network to capture only the most salient features of the data to perform reconstruction.

1.2 Linear Autoencoders and PCA Connection

If an under-complete autoencoder uses linear activation functions, the latent space span matches the subspace found by Principal Component Analysis (PCA). Introducing non-linear activations allows the autoencoder to model non-linear manifolds.

2

Over-Complete Autoencoders

2.1 Sparsity Penalties

An over-complete autoencoder has a latent bottleneck dimension larger than the input dimension. To prevent the model from learning a simple identity mapping, we apply a sparsity penalty (e.g. L1 regularization on the activations) to ensure only a small fraction of latent neurons fire for any given sample.

2.2 Denoising Autoencoders (DAE)

Denoising Autoencoders prevent identity mapping by corrupting the input data (adding noise or masking pixels) before passing it to the encoder. The model is trained to reconstruct the original, clean input, forcing it to learn the underlying structure of the data manifold.

2.3 Tied Weights Regularization

To reduce parameters and regularize the model, we can use tied weights, setting the decoder weight matrix to the transpose of the encoder weight matrix: Wdecoder = WencoderT.

3

Variational Autoencoders (VAEs)

3.1 Generative Latent Spaces

Standard autoencoders map inputs to discrete coordinates in latent space, which can result in gaps that produce gibberish when decoding. Variational Autoencoders (VAEs) resolve this by mapping inputs to probability distributions, ensuring a continuous and generative latent space.

3.2 Reparameterization Trick Math

To backpropagate through the random sampling step, we use the reparameterization trick. Instead of sampling directly from the latent distribution N(μ, σ2), we sample a random noise vector ε ∼ N(0, I) and calculate the latent vector as:

z = μ + σ ⌖ ε

This formulation shifts the stochastic component to an input node, allowing gradients to flow back through μ and σ.

4

VAE Loss Function

4.1 Reconstruction Error vs. KL Divergence

The VAE loss function contains two components:

  • Reconstruction Loss: Minimizes the difference between input and output, ensuring reconstruction quality.
  • KL Divergence Regularization: Measures how much the predicted latent distribution q(z|x) deviates from a standard normal prior p(z) = N(0, I), keeping the latent space structured.

4.2 Mathematical Balancing of Divergence

The total loss minimizes the negative Evidence Lower Bound (ELBO):

Loss = Eq(z|x)[ log p(x|z) ] - DKL( q(z|x) || p(z) )

Balancing these terms prevents the latent space from collapsing while maintaining reconstruction quality.
