Upsampling Techniques

1.1 Interpolation vs. Max Unpooling

Standard CNNs reduce spatial size (height/width) using pooling or strides. Generative tasks (like image generation, autoencoders, or semantic segmentation) require upsampling to reconstruct spatial details:

  • Nearest Neighbor / Bilinear Interpolation: Non-learnable techniques that fill in new pixel coordinates based on the nearest values or weighted averages of surrounding values.
  • Max Unpooling: Reconstructs shape by distributing activations back into the specific coordinates they originated from, using saved max pooling indices from the corresponding downsampling step. Other elements are filled with zero.

1.2 Transposed Convolution

Transposed Convolution (sometimes called Deconvolution) is a learnable upsampling operation. Instead of sliding a filter to calculate a weighted sum, we project a single input activation through a filter, distributing values over a larger output area. The weight matrix of this operation is mathematically the transpose of a normal downsampling convolution matrix.

Classical Vision Networks

2.1 AlexNet Architecture (2012)

AlexNet kick-started the modern deep learning era by winning the 2012 ImageNet competition with a 16.4% top-5 error rate:

  • Structure: 8 hidden layers (first 5 convolutional, last 3 fully connected).
  • Key Details: First large-scale use of ReLU activations, Dropout regularizers to prevent overfitting, and extensive Data Augmentation (crops, flips).

2.2 VGG Networks & Depth Limits

The Visual Geometry Group (VGG) established that network depth is a critical driver of representation capacity:

  • Key Innovation: Replaced large filters (like AlexNet's 11 × 11) with stacks of smaller 3 × 3 convolutional filters. Stacking two 3 × 3 filters yields the same receptive field as one 5 × 5 filter but uses fewer parameters and introduces more non-linear activations.
  • Depth: Scaled up to 19 layers (VGG-19) with 144 million parameters, achieving a 6.8% top-5 error rate.

Sequential Limits & Shattered Gradients

3.1 The Degradation Problem

For several years, researchers assumed that adding more layers would always improve performance. However, scaling sequential networks past ~20 layers caused classification accuracy to degrade. This degradation problem affects both training and test error, proving it is caused by training/optimization difficulties, not overfitting.

3.2 Shattered Gradient Phenomenon

In deep sequential networks, backpropagation gradients must pass through many layers. As depth increases, the gradient of the output with respect to the input changes rapidly with tiny changes in the input. The autocorrelation of gradients drops to zero. This is the shattered gradient problem: the loss landscape resembles a jagged range of tiny mountains rather than a smooth valley, making gradient descent extremely difficult.

4

Residual Connections

4.1 Residual Blocks & Skip Formulation

Residual Networks (ResNets) resolve the degradation problem by introducing parallel skip connections (residual branches) that add the input directly back to the output of a layer block:

y = x + f(x, φ)

Where x is the input, and f(x, φ) represents the convolutional transformations of the block. Instead of learning an entirely new output mapping, the layers only need to learn an additive change (residual) to the identity representation.

4.2 Skip Path Ensemble Interpretations

Unraveling the recursive equations of a ResNet reveals that it acts like an ensemble of networks of varying depths. A network with D residual blocks contains 2D potential computational paths from the input to the output. This ensemble structure prevents gradients from vanishing or shattering, keeping the loss landscape smooth.

5

ResNet Variance Explosion

5.1 Exponential Variance Growth

We use weight initialization methods (like He initialization) to keep activation variance constant across layers. However, in ResNets, adding the input x back to the block output f(x) causes a problem: since the signal from f(x) has uncorrelated variability, combining them increases the overall variance. The variance doubles at each residual block, growing exponentially (2D) with network depth D, leading to instability.

5.2 Scaling and Batch Norm Solutions

Two methods prevent this variance explosion in deep ResNets:

  1. Bilinear Scaling: Scaling the signal by 1 / √2 between each residual block keeps the variance constant, but degrades the identity representation of the skip connection.
  2. Batch Normalization placement (Standard): Placing a Batch Norm layer as the first step within the residual block, and initializing its scale γ to 1 and offset β to 0. This forces the input of each block to have unit variance. Under He initialization, the output variance of the block is also 1, causing the total variance to grow linearly with the number of blocks (1, 2, 3...) rather than exponentially. Additionally, at initialization, later layers are dominated by the skip connection, computing a clean identity function.
