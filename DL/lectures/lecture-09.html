Loss Landscapes

1.1 Convex Loss Landscapes

In simple machine learning models (like linear regression), the loss landscape is convex. A function is mathematically convex if every chord line segment between two points on the surface lies entirely above or on the function, never intersecting below it:

Convexity Properties
  • Single Minimum: Any local minimum is guaranteed to be the global minimum.
  • Convergence: Regardless of parameter initialization, standard gradient descent is mathematically guaranteed to reach the optimal parameters (global minimum) if the learning rate is chosen correctly.

1.2 Non-Convex Loss Landscapes

Deep neural networks introduce non-linear activation functions and hierarchical layer compositions. This renders their loss landscapes highly non-convex, introducing several challenging geometric features:

  • Local Minima: Valleys where the loss is lower than in the immediate surrounding area but higher than the global minimum.
  • Saddle Points: Points where the slope is zero, but the surface curves up in some directions and down in others (extremely common in high-dimensional neural network loss spaces).
2

First Order Optimization

2.1 Gradient Descent

Gradient descent is a first-order optimization method. It uses the partial derivatives of the loss function to update the model parameters in the direction of steepest descent:

φt+1 = φt - α ∇ J(φt)

Where φ represents the network parameters, α is the learning rate, and ∇ J(φt) is the gradient vector of the loss function.

2.2 Non-Convexity & Gabor Model

To study non-convex optimization, we analyze the Gabor model, which maps a scalar input to a scalar output using a sinusoidal function multiplied by a negative exponential component:

f[x, φ] = sin(φ0 x) × exp(-φ12 x2)

This model highlights how non-convex landscapes behave:

  • Starting Point Sensitivity: If initialized in the correct valley, gradient descent finds the global minimum. If initialized in a suboptimal valley, it descends to a local minimum.
  • Zero Gradients: If initialized exactly on a local maximum or saddle point, the gradient is zero, and the parameters cannot update.
3

Stochastic Gradient Descent

3.1 Introducing Randomness

Standard (batch) gradient descent evaluates the loss and gradient across the entire dataset before making a single parameter update, which is computationally prohibitive for large datasets. Stochastic Gradient Descent (SGD) solves this by introducing randomness:

  • Mini-batches: Updates are computed based on a randomly sampled subset of the training data (a batch) rather than the entire dataset.
  • Epochs: A single pass through all training examples (evaluating all batches without replacement) is called an epoch.

3.2 Mathematical Formulation & Pros

The SGD parameter update is computed as:

φt+1 = φt - α (1 / |Bt|) ∑i ∈ Bt ∇ ℓi(φt)

Where Bt contains the indices of the current batch, and ℓi is the loss of the i-th training sample.

Why SGD Outperforms Batch Gradient Descent
  • Computation: Calculating gradients over a small batch is much faster and requires significantly less memory.
  • Escaping Minima: Because the batch loss landscape is a noisy approximation of the global landscape, the gradient step can move uphill relative to the global loss. This noise allows the optimizer to escape shallow local minima.
  • Saddle Points: The stochastic noise breaks symmetry, preventing parameters from getting stuck near saddle points.

Momentum & Acceleration

4.1 Standard Momentum

SGD trajectories often suffer from heavy oscillations in steep, narrow valleys. To smooth this behavior, we introduce a momentum term inspired by physical dynamics:

mt+1 = β mt + ∇ J(φt)
φt+1 = φt - α mt+1

Where mt is the accumulated momentum velocity, and β ∈ [0, 1) is the momentum decay rate (typically 0.9). Momentum acts as an infinite weighted sum of past gradients, speeding up updates along consistent directions while canceling out oscillatory steps.

4.2 Nesterov Accelerated Momentum

Nesterov Accelerated Momentum (NAM) improves upon standard momentum by computing the gradient at a predicted "look-ahead" point instead of the current position:

mt+1 = β mt + ∇ J(φt - α β mt)
φt+1 = φt - α mt+1

Evaluating the gradient at the look-ahead position (φt - α β mt) acts as a braking mechanism: if the momentum is about to carry the parameters up an opposing slope, the look-ahead gradient points backward, dampening the oscillation ahead of time.

5

Adaptive Optimizers

5.1 RMSProp Optimizer

Gradient descent with a fixed learning rate makes large steps in directions with large gradients and tiny steps in directions with small gradients. RMSProp resolves this by scaling the learning rate for each parameter using an exponentially decaying average of its squared gradients:

vt+1 = β vt + (1 - β) (∇ J(φt))2
φt+1 = φt - [ α / √(vt+1 + ε) ] ⊙ ∇ J(φt)

Where vt is the moving average of squared gradients, β is the decay rate, and ε is a small constant (e.g., 10-8) to prevent division by zero. RMSProp discards distant history, localizing adaptation to the current loss landscape terrain.

5.2 Adam Optimizer

Adam (Adaptive Moment Estimation) combines the principles of Momentum (estimating the first raw moment) and RMSProp (estimating the second uncentered moment) with bias correction terms:

Adam Update Formulation
  1. Accumulate first moment (momentum):
    mt = β1 mt-1 + (1 - β1) gt
  2. Accumulate second moment (variance):
    vt = β2 vt-1 + (1 - β2) gt2
  3. Apply bias corrections (since m0, v0 are initialized to 0):
    m̂t = mt / (1 - β1t)  ,  v̂t = vt / (1 - β2t)
  4. Update parameters:
    φt = φt-1 - [ α / (√(v̂t) + ε) ] ⊙ m̂t
6

Second Order & Local Minima

6.1 Second Order Methods

Second-order optimization methods (like Newton's method) utilize curvature information from the second derivative (Hessian matrix H) to take optimal step sizes:

φt+1 = φt - H-1 ∇ J(φt)

However, second-order methods are rarely used in deep learning due to two main limits:

  1. Computational Complexity: For a network with N parameters, the Hessian matrix size is N × N. Storing and inverting this matrix costs O(N3) operations, which is completely impossible for modern models with millions or billions of parameters.
  2. Saddle Trap: Second-order methods are attracted to saddle points (where gradient is zero but curvature differs), whereas first-order adaptive methods easily navigate them.

6.2 The Myth of Local Minima

A common historical concern was that neural networks would get trapped in high-cost local minima. Modern high-dimensional research has shown this concern is largely a myth:

  • High Dimensions: For a point to be a local minimum, the loss surface must curve upwards in every single parameter direction. In high dimensions, the probability of this occurring at high loss levels is virtually zero.
  • Saddle Points are Dominant: Instead of local minima, the primary optimization obstacles are saddle points, which have zero gradient but directions of downward curvature that first-order noise-based optimizers can exploit.
  • Good Enough Minima: Almost all local minima that do exist are clustered close to the bottom of the loss landscape, yielding similar, highly-optimal generalization performance.
