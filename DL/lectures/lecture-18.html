Attention Is All You Need

1.1 Dispensing with Recurrence

In 2017, Vaswani et al. published the paper "Attention Is All You Need", introducing a radical architectural shift: the Transformer. Standard sequence models (RNNs, LSTMs, GRUs) rely on recurrence, processing tokens sequentially one-by-one. The Transformer dispenses with recurrence entirely, relying solely on self-attention mechanisms to model relationships between tokens in a single step.

1.2 Massive Parallelization Benefits

Because RNNs must process tokens sequentially, they cannot be parallelized during training—calculating hidden state ht requires waiting for ht-1. The Transformer processes all tokens in a sequence simultaneously. This massive parallelization allows models to train significantly faster on modern GPU hardware, enabling the training of Large Language Models (LLMs) on massive datasets.

2

Query, Key, and Value Vectors

2.1 Database analogy QKV mapping

Self-attention operates similarly to a database retrieval system. Each input token embedding is projected into three distinct vectors using trained weight matrices (WQ, WK, WV):

  • Query (Q): Represents the current token searching for related context.
  • Key (K): Represents what information a token holds, matching against queries.
  • Value (V): Contains the actual semantic content of the token, retrieved when queries and keys match.
3

Scaled Dot-Product Attention

3.1 Attention Equation

To calculate self-attention across an entire sequence, we bundle the Q, K, and V vectors into matrices and compute Scaled Dot-Product Attention:

Attention(Q, K, V) = softmax( [ Q KT / √(dk) ] ) V

Where Q KT calculates raw similarity scores between all queries and keys, and dk is the dimensionality of the key vectors.

3.2 The Scaling Factor derivation

If key dimension dk is large, the dot product values grow large. Passing these large values through a softmax function concentrates the probability distribution, leading to extremely small gradients (vanishing gradients) during training. Scaling the dot products by 1 / √(dk) keeps the variance of the dot products at 1, preventing softmax saturation and ensuring stable training.

4

Multi-Head Attention Blocks

4.1 Parallel representation spaces

Instead of performing a single attention pass, the model projects Q, K, and V vectors into multiple lower-dimensional representation subspaces in parallel. This is Multi-Head Attention (MHA). Each "head" focuses on different types of relationships (e.g. one head links subject-verb agreements, while another associates pronouns). The outputs of all heads are concatenated and projected back to the original dimension.

4.2 Encoder vs. Decoder Blocks

The standard Transformer architecture consists of stacked blocks:

  • Encoder Block: Multi-Head Self-Attention + Position-wise Feed Forward Network (FFN). It uses residual skip connections and Layer Normalization (LayerNorm) around each sub-layer.
  • Decoder Block: Similar to the encoder but contains a third sub-layer:
    • Masked Multi-Head Self-Attention: Masks future tokens during training (setting attention scores to -∞ before softmax) to maintain causality during autoregressive generation.
    • Encoder-Decoder Cross-Attention: Computes queries Q from the decoder, while keys K and values V come from the encoder's output representations, aligning the target sequence with the source.

Positional Encoding & Context

5.1 Sine and Cosine coordinate injections

Since the Transformer processes all tokens simultaneously, it has no inherent sense of word order. To address this, we inject positional information directly into the input embeddings using Positional Encodings (PE). We compute coordinate offsets using sine and cosine functions of different frequencies:

PE(pos, 2i) = sin( pos / 100002i/dmodel )
PE(pos, 2i+1) = cos( pos / 100002i/dmodel )

Where pos is the token's position and i is the dimension index. Adding these vectors to the word embeddings allows the model to learn relative distances between tokens.

5.2 Contextual enrichment of embeddings

Self-attention dynamically updates static word embeddings. In models like BERT and GPT, the representation of a word (e.g. "bank") changes dynamically depending on the other tokens in the sentence. This contextual enrichment resolves polysemy, allowing the model to represent different meanings of the same word in vector space.
