Positional Encoding Mechanics

1.1 Sine/Cosine Curves Injection

Instead of adding a single incrementing integer (which could grow infinitely for long sentences and dominate the input), Transformers use sine and cosine functions of different frequencies to generate unique positional signatures:

PE(pos, 2i) = sin( pos / 100002i/dmodel )
PE(pos, 2i+1) = cos( pos / 100002i/dmodel )

This formulation creates a geometric progression of wave frequencies, allowing the model to represent both absolute positions and relative distances between tokens.

1.2 Vector Addition Advantages

We combine the positional encoding directly with the word embedding via vector addition: vfinal = vword + PEpos. In a high-dimensional space (e.g. 512 dimensions), adding these vectors does not destroy the semantic information of the word. The model can easily learn to project the vector into orthogonal subspaces, extracting the semantic meaning and the position watermark independently.

2

Layer Normalization

2.1 Internal Covariate Shift

As activations pass through numerous deep layers of matrix multiplications, their scale can shift dramatically (vanishing/exploding activations). This change in activation distributions is known as internal covariate shift, which slows down training and destabilizes learning.

2.2 LayerNorm vs. BatchNorm

While Batch Normalization normalizes activations across a batch of samples for each feature independently, Layer Normalization normalizes activations across all features of a single word vector. This makes LayerNorm independent of the batch size and highly effective for processing variable-length sequences, making it standard for Transformers.

2.3 Step-by-Step Numerical Walkthrough

Let's walk through LayerNorm for a single word vector x = [3.0, 5.0, 2.0, 8.0]:

  1. Calculate Mean (μ): μ = (3 + 5 + 2 + 8) / 4 = 18 / 4 = 4.5
  2. Calculate Variance (σ2): σ2 = [ (3 - 4.5)2 + (5 - 4.5)2 + (2 - 4.5)2 + (8 - 4.5)2 ] / 4 = [ 2.25 + 0.25 + 6.25 + 12.25 ] / 4 = 21 / 4 = 5.25. (Standard Deviation σ ≈ 2.291)
  3. Normalize: x̂ = (x - μ) / √(σ2 + ε). For x1: (3 - 4.5) / 2.291 ≈ -0.654
  4. Scale and Shift: y = γ x̂ + β. Using learnable vectors γ (initialized to ones) and β (initialized to zeros) to yield the normalized output.
3

Encoder Architecture

3.1 Tokenization & Embeddings

Before entering the encoder, the raw sentence is split into tokens (subwords). Each token is mapped to a static 512-dimensional vector embedding, representing its basic lexical identity.

3.2 Stacking Encoder Blocks

The standard Transformer stack contains 6 identical encoder blocks. The output vectors from one block become the input vectors for the next block, incrementally refining the context of each word vector.

3.3 Attention and Normalization Pipelines

Each block contains two main sub-layers: a Multi-Head Attention layer and a Position-wise Feed-Forward network. Both sub-layers are wrapped in residual skip connections followed by Layer Normalization (often referred to as the Add & Norm step).

4

Feed-Forward Networks in Encoder

4.1 Two-Layer MLP

Each encoder block features a Position-wise Feed-Forward Network (FFN) applied to every token vector independently and identically. It consists of two linear transformations with a ReLU activation in between:

FFN(x) = max(0, x W1 + b1) W2 + b2

4.2 Dimensionality Expansion-Contraction

The FFN acts as a projection tunnel: the first layer expands the token vector's dimension from 512 to 2048, allowing the model to learn complex non-linear combinations. The second layer contracts it back to 512 dimensions to match the residual path.

5

Training vs. Inference: Teacher Forcing

5.1 Autoregressive Inference

During inference (generation), the model generates tokens one-by-one. Each generated token is appended to the input sequence to predict the next token. If the model makes an error early in the generation, that error cascades, affecting all subsequent predictions.

5.2 Teacher Forcing in Parallel Training

To avoid slow sequential processing during training, we use Teacher Forcing. Since the ground truth target sequence is already known, we pass the entire correct target sequence into the decoder at once, calculating the gradients for all token positions in parallel.

5.3 The Danger of Data Leakage

Passing the complete target sequence to the decoder simultaneously presents a risk: the self-attention mechanism at token 1 would naturally inspect tokens 2, 3, and 4 (the future). This data leakage allows the model to cheat during training, rendering it incapable of performing actual step-by-step generation during inference. We solve this using causal masking.
