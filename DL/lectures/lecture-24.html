CNN Limitations vs. ViT

1.1 Inductive Bias vs. Global Context

CNNs rely on strong inductive biases: translation invariance and local receptive fields (pixels close together are related). While this is efficient for training on small datasets, it restricts CNNs from modeling long-range global relationships. Vision Transformers (ViTs) drop these inductive biases, allowing the model to learn global context from the first layer using self-attention.

1.2 Computational Complexity Comparison

CNN convolution scales linearly with the number of pixels. Self-attention scales quadratically with sequence length, which presents a challenge when translating images to token sequences since an image has a large number of pixels.

2

Vision Transformer (ViT) Architecture

2.1 Patch Extraction & Linear Projection

To apply Transformers to images, ViT splits an image into a grid of non-overlapping patches (e.g. 16×16 pixels). Each patch is flattened and projected into a vector embedding using a linear layer, treating image patches similarly to word tokens.

2.2 The [CLS] Token

Following the BERT architecture, a learnable class [CLS] token is prepended to the patch embeddings. The final state of this token serves as the aggregate representation for image classification tasks.

2.3 Positional Embeddings in Images

To preserve spatial location, 1D learnable positional embeddings are added to the patch vectors, allowing the Transformer blocks to resolve spatial relationships.

3

DeiT (Data-efficient Image Transformers)

3.1 Distillation Tokens

ViT models require massive datasets (like JFT-300M) to train because they lack CNN inductive biases. DeiT solves this by using a new distillation token alongside the [CLS] token, allowing the model to learn efficiently from smaller datasets (like ImageNet-1k).

3.2 Teacher-Student Training for Vision

The distillation token interacts with the patch tokens and [CLS] token via self-attention, learning to match the predictions of a strong teacher CNN, which injects inductive biases into the Transformer model.

4

CLIP (Contrastive Language-Image Pretraining)

4.1 Joint Image-Text Embeddings Space

CLIP trains an image encoder and a text encoder to project both modalities into a shared embedding space. It aligns descriptions with corresponding images by maximizing their cosine similarity.

4.2 Contrastive Loss Matrix

For a batch of N image-text pairs, the model calculates an N × N similarity matrix. The contrastive loss maximizes the similarity of the N correct pairs along the diagonal while minimizing the similarity of the N2 - N incorrect pairs off the diagonal.

4.3 Zero-Shot Inference

By comparing an image embedding against text embeddings of target labels (e.g. "a photo of a cat", "a photo of a plane"), CLIP performs zero-shot classification without training on specific downstream target datasets.

Introduction to Autoencoders

5.1 The High-Dimensionality Bottleneck

Real-world data exists in high-dimensional spaces, but the underlying information often lies on a lower-dimensional manifold. Autoencoders find this manifold by compressing inputs into a bottleneck layer before reconstruction.

5.2 Reconstruction Loss Function

We train autoencoders to minimize reconstruction loss (the difference between the input x and reconstructed output x̂) using Mean Squared Error (MSE) or Binary Cross-Entropy:

L(x, x̂) = ||x - x̂||2

5.3 Autoencoders vs. PCA

PCA is limited to linear projection directions. Autoencoders use non-linear activation functions in their networks to learn complex non-linear coordinate mappings, projecting data onto complex manifolds.
