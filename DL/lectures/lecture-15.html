Gradient Clipping & BiRNNs

1.1 Exploding Gradients & Clipping

To prevent gradients from growing exponentially (exploding gradient problem) during BPTT on long sequences, we apply Gradient Clipping. When the norm of the gradient vector g exceeds a predefined threshold c, we rescale the gradient before updating weights:

if ||g|| > c:   g = c × [ g / ||g|| ]

This operation scales down the gradient magnitude to fit within the threshold while keeping the direction of descent unchanged, ensuring training stability.

1.2 Bidirectional Recurrent Nets (BiRNN)

Standard RNNs process sequences in chronological order (left-to-right), meaning the hidden state at step t only contains context from the past. However, tasks like translation or word disambiguation often require future context (e.g. classifying the word "bank" in "He sat by the bank to watch the water" vs. "He went to the bank to withdraw money"):

  • Structure: A Bidirectional RNN (BiRNN) uses two separate hidden layers. One layer processes the sequence forward (past context), and the other processes the sequence backward (future context). Their outputs are concatenated at each step.
  • Streaming Limitation: Since the backward pass requires reading the entire input, BiRNNs cannot be deployed for real-time streaming tasks where the future is unavailable.

LSTM Gate Mechanics

2.1 Forget and Input Gates

The Long Short-Term Memory (LSTM) network (Hochreiter & Schmidhuber, 1997) addresses the vanishing gradient problem by separating memory storage from output generation. LSTMs regulate information flow using multiplicative gates:

  • Forget Gate (ft): Decides what information to discard from the long-term memory (cell state). Outputs values between 0 (forget completely) and 1 (keep completely) using a sigmoid activation:
    ft = σ(Wf [ht-1, xt] + bf)
  • Input Gate (it): Decides which new features to write into the cell state:
    it = σ(Wi [ht-1, xt] + bi)

2.2 Candidate and Output Gates

  • Candidate Cell State (C̃t): Computes new candidate values to add to the memory using a tanh activation:
    C̃t = tanh(Wc [ht-1, xt] + bc)
  • Output Gate (ot): Decides what information to read from the cell state to produce the short-term memory (hidden state ht):
    ot = σ(Wo [ht-1, xt] + bo)
    ht = ot ⊙ tanh(Ct)
3

LSTM Memory Architecture

3.1 Cell State Information Superhighway

The core innovation of the LSTM is the cell state (Ct), which runs linearly through the entire sequence chain. The cell state update is computed as:

Ct = ft ⊙ Ct-1 + it ⊙ C̃t

Where ⊙ represents element-wise (Hadamard) multiplication. Because the update is additive rather than multiplicative, backpropagating errors along the cell state does not require repeated matrix multiplications by Whh. The cell state acts as an information superhighway, allowing gradients to flow backward through time without vanishing.

Gated Recurrent Units (GRU)

4.1 Reset and Update Gates

Introduced by Cho et al. (2014), the Gated Recurrent Unit (GRU) is a simplified alternative to the LSTM. It merges the cell state and hidden state into a single state ht and uses only two gates:

  • Reset Gate (rt): Decides how much of the past hidden state to forget when computing the new candidate state:
    rt = σ(Wr [ht-1, xt] + br)
  • Update Gate (zt): Acts as a combined forget and input gate, determining how much of the previous state ht-1 to keep versus how much new content to write:
    zt = σ(Wz [ht-1, xt] + bz)

4.2 GRU Candidate State & Updates

The candidate state and hidden state updates are computed as follows:

GRU Equations
  • Candidate State (h̃t): Applies the reset gate to the previous hidden state:
    h̃t = tanh(Wh [rt ⊙ ht-1, xt] + bh)
  • Hidden State Update (ht): Linearly interpolates between the previous state and candidate state using the update gate:
    ht = (1 - zt) ⊙ ht-1 + zt ⊙ h̃t
5

LSTM vs. GRU Comparison

5.1 Parameter & Computation Trade-offs

When selecting between LSTMs and GRUs, engineers evaluate the following trade-offs:

  • Parameters & Compute: GRUs have only two gates, while LSTMs have three. GRUs use fewer parameters (roughly 3/4 the weight matrices of LSTMs), making them faster to train and computationally cheaper.
  • Capacity: LSTMs maintain separate short-term and long-term memory states, which can perform better on large datasets with complex, long-range dependencies.
  • Heuristic Rule: Start with LSTM as the default choice. If training speed, parameter constraints, or hardware limits are key, switch to GRUs. Both models yield similar performance in practice.
