Backpropagation Principles

1.1 Computational Graphs & Chain Rule

Backpropagation is the core algorithm used to compute gradients of the loss function with respect to the network weights and biases. It represents mathematical equations as a computational graph where nodes are operations (e.g., addition, multiplication) and edges are variables. By applying the chain rule of calculus, the gradient of the final output (loss) is backpropagated through intermediate nodes:

∂L / ∂x = (∂L / ∂y) × (∂y / ∂x)

Where y is an intermediate variable calculated from x, and L is the loss.

1.2 Forward vs. Backward Pass

The calculation of gradients involves two sequential passes:

  • Forward Pass: We input data and compute activations layer-by-layer to calculate intermediate node values and the final loss.
  • Backward Pass: We start at the final loss and move backward, calculating local derivatives at each node and multiplying them by incoming gradients from downstream nodes. This pass evaluates parameter gradients efficiently in O(N) operations.

Parameter Initialization

2.1 The Need for Symmetry Breaking

Choosing proper initial parameter values is critical for optimization convergence:

  • Zero Initialization Failure: If we initialize all weights to zero, every hidden unit in a layer receives the exact same inputs and computes the same outputs. Consequently, their gradients are identical, and they update in the same way, failing to break symmetry and represent complex features.
  • Random Initialization: Small random values break symmetry but risk vanishing or exploding activations/gradients in deep architectures.

2.2 Xavier vs. Kaiming/He Methods

Standard initialization strategies scale initial weights by the number of inputs (nin) and outputs (nout) to keep activation variance constant across layers:

Method Target Activation Weight Variance (σ2) Core Idea
Xavier (Glorot) Sigmoid / Tanh 2 / (nin + nout) Balances input/output scaling. Assumes linear activation regions.
Kaiming (He) ReLU / Leaky ReLU 2 / nin Compensates for ReLU setting half of the activations to 0.

Double Descent Phenomenon

3.1 MNIST-1D Experimental Observations

Classical statistical learning theory dictates a U-shaped generalization curve where test error increases as model capacity exceeds the dataset complexity (overfitting). However, modern deep learning exhibits Double Descent:

  • Standard Clean Data: Test error drops, spikes near the point where the model capacity exactly matches the number of training samples, and then drops again in the highly over-parameterized regime.
  • Noisy Labels (15% Noise): The double descent effect is highly pronounced. The test error climbs to a massive peak at the interpolation threshold before dropping to a new, lower global minimum.

3.2 The Inductive Bias in Over-parameterization

Once model capacity is large enough to drive training loss to near zero (the interpolation threshold), the model fits all training points perfectly. As we add more parameters, the training error remains at zero, but the model has extra capacity to configure the spaces between data points. Its inductive bias forces it to select the smoothest possible interpolating function, reducing test variance and improving generalization.

Generalization Regimes

4.1 Three Regimes of Model Capacity

Double descent divides the parameter/capacity spectrum into three distinct regimes:

Capacity Regimes Summary
  • Under-parameterized Regime: Classical statistical zone. Adding parameters increases representation capacity, reducing bias and lowering test error.
  • Critical Regime: The interpolation threshold. The model has just enough capacity to fit the training set, meaning it must warp its function to fit noise, leading to high variance and peak test error.
  • Over-parameterized Regime: The deep learning zone. The model has far more parameters than training samples. Optimization methods select smooth interpolations, reducing variance and yielding superior test performance.

Error Analysis & Workflow

5.1 Noise, Bias, and Variance Errors

Generalization error is decomposed into three components:

  • Noise (Irreducible Error): Inherent uncertainty in the task or dataset (e.g. overlapping classes). It cannot be removed by optimization.
  • Bias: Systematic error due to incorrect model assumptions. Can be reduced by increasing model size or depth.
  • Variance: Sensitivity to variations in the specific training dataset. Can be reduced by adding more data, applying regularization, or moving deeper into the over-parameterized regime.

5.2 Hyperparameter Splits & Search

Hyperparameters cannot be optimized via gradient descent. To search the parameter space, we split data into three separate partitions:

  • Training Set: Used to compute gradients and optimize network parameters (weights/biases).
  • Validation Set: Used to evaluate different hyperparameter settings (learning rates, layer sizes) and select the best candidate configuration.
  • Test Set: Kept hidden until the end of all developments to report final, unbiased generalization metrics.
