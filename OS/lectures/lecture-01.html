Introduction to Operating Systems

1.1 The Mystery of Saving a File

What actually happens when you click "Save" in an application like Word? The application hands the data over to the Operating System, which coordinates with the hardware (disk/SSD) to safely store your data.

1.2 Why Study Operating Systems?

The OS is the foundation of all software. Understanding it benefits every programmer:

  • Performance: Understand why some programs are fast and others are slow.
  • Reliability: Grasp the reasons behind crashes and failures.
  • Security: Learn how systems protect vital resources.
  • Career: Systems knowledge is highly demanded in the industry.
Definition: Operating System

An Operating System fulfills three core roles:

  • Resource Manager: Allocates CPU, memory, and I/O devices.
  • Control Program: Supervises execution and prevents errors.
  • Extended Machine: Simplifies complex hardware for software to use.
Analogy: The City Infrastructure

If a Computer is a City, the OS is its Infrastructure:

  • Roads/Highways = System buses
  • Power Grid = Power management
  • Police/Courts = Security mechanisms
  • Postal Service = Inter-process communication (IPC)
  • Zoning Board = Memory management

Without an OS, hardware is just expensive silicon.

Checkpoint Question

Is a web browser an Operating System?

Answer: No. Even though Chrome manages processes, memory, and security internally, browsers use OS services; they do not provide them to other applications.

History of Operating Systems

2.1 The 1950s: Batch Processing

The dawn of computing involved programs on punch cards and manual job queues managed by operators. There was no user interaction during execution, and the CPU was idle most of the time.

First OS: Automated batch processing monitors.

2.2 The 1960s: Time-Sharing Revolution

Multics (Multiplexed Information and Computing Service) revolutionized computing by allowing multiple users on one computer simultaneously. Rapid switching gave the illusion of a dedicated machine.

Legacy of Multics

It introduced concepts we still use today: memory protection, file permissions, and virtual memory.

2.3 The 1970s-1980s: UNIX Era & Personal Computing

  • UNIX Evolution: UC Berkeley created BSD, focusing on portability and simplicity. Modern macOS inherited BSD DNA.
  • Personal Computing: The Apple II and Commodore 64 appeared. In 1984, the Macintosh brought the GUI to the masses.

2.4 The 1990s: NeXTSTEP

Steve Jobs' NeXT Computer combined a Mach microkernel with BSD UNIX and an elegant object-oriented interface. Tim Berners-Lee invented the WWW on a NeXT machine. Apple's acquisition of NeXT in 1996 formed the foundation of modern macOS.

2.5 The 2000s-Present: Modern Era

  • 2001: Mac OS X launched (UNIX-certified).
  • 2007: iOS brings OS X to mobile devices.
  • 2017: APFS replaces HFS+.
  • 2020: Apple Silicon transition.

The 5 Core OS Services

An Operating System provides five fundamental services to manage the hardware effectively.

3.1 Process Management

Manages hundreds of programs simultaneously through isolation and efficient resource sharing.

  • Process creation and termination (fork, exec, wait).
  • CPU scheduling across multi-core processors.
  • Context switching (rapidly swapping processes in and out of the CPU).

3.2 Memory Management

Creates the "Infinite Memory Illusion" allowing hundreds of processes to feel like they each use gigabytes of memory, even on a 16GB RAM machine.

  • Virtual memory per process.
  • Physical RAM sharing and protection.
  • Paging and swapping to disk.
  • Memory compression (common on macOS).

3.3 File Systems

Organizes persistent storage so billions of blocks can be organized into files you find instantly.

  • Modern File Systems (like APFS) offer Copy-on-write, snapshots, and encryption.
  • Maintains hierarchical directories.
  • Enforces permissions and access control.
  • Manages file metadata.

3.4 I/O Management

Provides a standardized interface to abstract thousands of different hardware device types.

  • Device drivers (e.g., IOKit on macOS).
  • Handling interrupts vs polling vs DMA.
  • Data buffering and caching.
  • Support for hot plugging devices.

3.5 Security

Acts as the guardian of the system. Security requires OS-level enforcement to be effective.

Important Security Features
  • Process isolation (separate memory spaces).
  • System Integrity Protection (SIP).
  • Sandboxing (e.g., App Store apps).
  • Gatekeeper and code signing.
  • Hardware integrations like Secure Enclave.
4

System Calls & Modes

4.1 Services Working Together

A simple action like saving a file requires complex orchestration across all services:

  1. Process Management: Handles the click event.
  2. Security: Checks application permissions.
  3. Memory: Copies data from application memory.
  4. File System: Allocates space, updates metadata.
  5. I/O: Writes the data to the SSD.
  6. Returns success to the application.

4.2 System Calls - The Bridge

Definition: System Call

The programming interface that applications use to request services from the OS kernel.

  • Files: open(), read(), write(), close()
  • Processes: fork(), exec(), wait()
  • Memory: mmap(), munmap()
  • Communication: pipe(), socket()

The POSIX standard makes these system calls portable across UNIX systems.

4.3 User Mode vs Kernel Mode

Modern processors utilize two distinct privilege levels to ensure stability, security, and abstraction. A user mode application cannot access hardware directly, read other processes' memory, or execute privileged instructions.

Mode Transitions

Transitions between user and kernel mode occur via:

  • System calls (intentional requests).
  • Interrupts (hardware events).
  • Exceptions (errors like page faults).

Note: A context switch takes roughly 2-5 microseconds of overhead.

4.4 Kernel Panic

A kernel panic is a total system failure where the system forcefully halts instead of trying to recover.

Why panic instead of recover?

If the kernel's internal state becomes suspect, continuing to run risks severe data corruption. Causes include defective hardware, bad kernel extensions (KEXTs), driver issues, or file system corruption.

5

Kernel Architecture

5.1 Monolithic Kernel

All OS services run in one single Kernel Space. Example: Linux.

  • Advantages: Very fast (direct function calls), efficient communication between services.
  • Disadvantages: One bug in any component can crash the entire system, large attack surface, highly complex (30M+ lines of code).

5.2 Microkernel

A minimal kernel that only provides fundamental services. Other services run in User Space. Examples: Original Mach, QNX, MINIX.

  • Provides only: Memory management, IPC (message passing), and basic scheduling.
  • Advantages: High robustness, better security, modularity.
  • Disadvantages: Performance overhead due to constant message passing (IPC) between user space and kernel space.

5.3 Hybrid Approach (macOS XNU)

macOS uses the XNU kernel ("X is Not Unix"), aiming for the "Best of Both Worlds".

XNU Architecture
  • Mach microkernel core: Handles memory, IPC, threads.
  • BSD layer: Handles POSIX APIs, networking, and file systems.
  • IOKit: Object-oriented device drivers (C++ based).

Result: Microkernel modularity and structure combined with monolithic performance.

Summary & Resources

Quick Revision: Key Takeaways
  • OS Core Purpose: Resource Manager, Control Program, and Extended Machine.
  • 5 Core Services: Process, Memory, File Systems, I/O, and Security.
  • System Calls: The bridge between user applications and kernel privileges.
  • Architecture: Monolithic (Fast but fragile) vs Microkernel (Secure but slower) vs Hybrid (macOS XNU).

Exam-Focused FAQ

Q: Why does macOS use a Hybrid Kernel?

A: To achieve the modularity and stability of a microkernel (Mach) while maintaining the raw performance of a monolithic kernel for high-traffic subsystems like networking and file systems (BSD).
