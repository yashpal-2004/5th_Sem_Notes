Why CPU Scheduling is Essential

1.1 The Reality

Your laptop right now has 200+ processes competing for CPU time, but only 4-8 cores to run them. Without scheduling, opening a text file might have to wait for a video to finish encoding, and mouse clicks would lag severely.

With Smart Scheduling: We get responsive interfaces, fair resource distribution, optimized waiting times, and predictable behavior.

1.2 The Three Ws

  • Where is it done? The Ready Queue.
  • Who did it? The Short-Term Scheduler (or CPU Scheduler).
  • When? Based on the Scheduling Criterion (Preemptive or Non-Preemptive).
2

The Coffee Shop Analogy

2.1 The Constraint

Imagine a single Barista trying to serve a line of Customers. The goal is to serve everyone as fast as possible, but the constraint is that the Barista can only make one drink at a time (1 CPU).

If Customer A orders 10 lattes (takes 10 mins), and Customer B orders 1 espresso (takes 1 min), what is the fairest way to serve them? This is the core challenge of CPU Scheduling.

3

FCFS: First Come, First Served

3.1 The "Wait Your Turn" Algorithm

Core Principle: Serve processes in exact arrival order. No cutting in line. Once started, the process runs to completion (Non-Preemptive).

Uses a FIFO Queue (First-In, First-Out).

3.2 Calculating Times

Formulas:

  • Turnaround Time (TAT) = Completion Time (CT) - Arrival Time (AT)
  • Waiting Time (WT) = Turnaround Time (TAT) - Burst Time (BT)

3.3 The Convoy Effect

The Convoy Effect Problem

One long process holds up many short processes. The average waiting time increases drastically. This is very poor for interactive/responsive systems.

4

SJF: Shortest Job First

4.1 Core Principle

The "Non Preemptive Minimum Execution Time" Algorithm. Select the process with the shortest next CPU burst. Shortest tasks cut to the front of the line.

Mode: Non-Preemptive. Once a job starts, it finishes before the next shortest is picked.

Data Structure: Priority Queue (Min-Heap).

Pros: Provably optimal for minimizing average waiting time.

4.2 The Starvation Problem

Starvation

If a continuous stream of shorter jobs keeps arriving, a longer job will sit in the queue indefinitely and never get executed. It "starves" waiting for CPU time.

5

SRTF: Shortest Remaining Time First

5.1 Preemptive SJF

SRTF evaluates the queue at every arrival and completion. If a new job arrives with less time than what is left on the current running job, the OS preempts (stops) the current job and switches to the new one.

Benefit: The Convoy Effect is completely eliminated.

5.2 macOS Reality (MLFQ)

Pure SJF and SRTF require knowing the exact burst time of every process in advance. Reality: The OS cannot predict the future!

The Solution

Modern systems like macOS use Multilevel Feedback Queues (MLFQ). This acts as a smart guessing system that uses past behavior to dynamically adjust priorities based on CPU usage patterns, giving us 80-90% of SJF's benefits without needing to know the future.

6

Real-World Applications

6.1 Where macOS uses FCFS and SJF

FCFS:

  • Print Queues (Printer Spooler) - Documents print in the order requested.
  • Audio Processing Pipeline - Audio chunks process sequentially for seamless playback.
  • File Write Operations - Sequential logging to disk.

SJF (Quality of Service - QoS):

macOS uses QoS classes as educated guesses about task duration to simulate SJF:

  • userInteractive → assumed very short → high priority
  • background → assumed very long → low priority
