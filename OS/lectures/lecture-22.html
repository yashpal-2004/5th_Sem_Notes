Jack Compiler & Language Syntax

3.1 Machine Language (Layer 0)

The "Ground Floor" of software.

Binary Instructions are 16-bit strings such as:

0000000000000010 // A-instruction
1110110000010000 // C-instruction

Characteristics: Directly executed by hardware, extremely tedious for humans, and provides zero abstraction. It is the "DNA" of the computer.

3.2 Assembly Language (Layer 1)

Symbolic Representation: @sum instead of @16.

The Assembler’s Job:

  • Translate symbols to addresses.
  • Convert mnemonics (e.g., D=M) to binary.
  • Resolve labels and output .hack files.
Analogy

Using symbolic labels (like @sum) in assembly is like using a contact name (e.g., "Mom") in your phone contact list instead of memorizing her 10-digit phone number.

3.3 Virtual Machine (Layer 4)

Intermediate Abstraction: A stack-based model.

VM Commands: push constant 5, add, pop local 0.

Why VM?

  • Platform Independence: Write once, run on any hardware with a VM translator.
  • Simplifies Compiler: The Jack compiler only needs to know how to translate high-level constructs into VM instructions.

3.4 The Jack Compiler (Layer 5)

Compilation Pipeline:

  • Tokenizer: Breaking text into "words" (keywords, symbols).
  • Parser: Understanding the "grammar" (syntax tree).
  • Code Generator: Emitting the VM instructions.

Primitive Types & Scope

4.1 Jack Language (Layer 6)

High-level, Object-Oriented.

Features: Classes, methods, fields, and static variables.

Role: The bridge between human logic and the VM translator.

class Main {
    function int addPair(int x, int y) {
        var int result;
        let result = x + y;
        return result;
    }
}

4.2 The Operating System (Layer 6)

Jack OS Library: 8 Core Modules.

Math.jack (Mult/Div), Output.jack (Text), Screen.jack (Graphics), Memory.jack (Heap), String.jack, Keyboard.jack, Sys.jack The "Library" Model: In Nand2Tetris, the OS is a set of functions called directly by your program—no complex kernel/user mode separation.

4.4 Jack vs Java/C

Similarities: Statically typed, OO structure, curly braces.

Differences: No inheritance, no interfaces, very simple memory management, no garbage collection (in the basic version).

5

Let & Do Statements

5.1 Jack File Structure

Rule: One class per file. Filename must match ClassName.

Structure: Class declaration -> Fields/Statics -> Constructor -> Subroutines.

5.2 Primitive Data Types

int: 16-bit signed (-32768 to 32767). char: ASCII character. boolean: true or false.

Arrays & Strings: Handled as objects using Array.new(size) and String.new(length).

5.3 Variable Types & Scope

  • Field: One per object instance (State).
  • Static: Shared by all instances of a class.
  • Local: Exists only during subroutine execution (Stack).

5.4 Memory Management (Jack Style)

Heap Management is handled dynamically by the Memory.jack OS module:

  • Allocation: 
    let arr = Array.new(10); // Allocates a block of size 10 on heap
  • Deallocation: 
    do arr.dispose(); // Manually frees the allocated block

Jack Operating System Library

6.1 Basic Statement: let and do

  • let: Used for assignments. 
    let x = 5;
  • do: Used for subroutine calls when you ignore the return value. 
    do Screen.drawPixel(x, y);

6.2 Control Flow: If-Else

Conditions must be boolean expressions.

Syntax: if (condition) { ...
} else { ... }.

6.3 Control Flow: While

The only loop construct in Jack!

// Syntax:
while (condition) {
    ...
}

// Example:
let sum = 0;
let i = 0;
while (i < length) {
    let sum = sum + arr[i];
    let i = i + 1;
}

// Display the result
do Output.printString("Sum of array elements: ");
do Output.printInt(sum);
do Output.println();

6.5 Steps

Steps to compile and execute Jack software suite programs:

  1. Create your source files with the .jack extension.
  2. Translate your high-level Jack code into VM language by running the compiler script in your tools directory: 
    ~/Desktop/nand2tetris/tools/JackCompiler.sh <folder-path>
  3. Open the VM Emulator: 
    ~/Desktop/nand2tetris/tools/VMEmulator.sh
  4. Select "Load Program" (or Load Folder) and choose the directory containing your compiled .vm files.
  5. Set animation speed to "Fast" or "No Animation" for complex programs/games.
  6. If logic errors occur, revise .jack files, recompile, and reload program.
