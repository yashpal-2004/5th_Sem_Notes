Quick Review

1.1 FCFS, SJF, SRTF Flaws

Previously, we saw that:

  • FCFS suffers from the Convoy Effect.
  • SJF suffers from Starvation (long processes never run).
  • SRTF fixes the Convoy Effect but introduces High Overhead (too many context switches).
The Conclusion

All three of these algorithms are not truly fair to all processes.

2

Round Robin (RR)

2.1 The "Everyone Gets a Turn" Algorithm

Core Principle: Each process gets a small unit of CPU time called a time quantum (q). After the quantum expires, the process is preempted and goes to the back of the ready queue. The CPU scheduler then picks the next process.

  • Criteria: Time Quantum (q).
  • Mode: Preemptive (Timer interrupt forces the switch).
  • Data Structure: Circular FIFO Queue.

2.2 The Quantum Effect

Choosing the correct quantum size is the key to Round Robin:

  • Too Small (e.g., 1ms): High Context Switch Overhead. The barista spends more time switching cups than making coffee!
  • Too Large (e.g., 10 mins): It degrades into FCFS. Customer A hogs the barista while Customer B waits forever.
  • Perfect Quantum: Usually 10-100ms. This keeps context switch overhead < 1% of the quantum while providing a good interactive response.

2.3 macOS Implementation

When you run Safari, Spotify, and VS Code, macOS uses a Round Robin-style approach within the same priority level. It gives each app a tiny slice of time so they all feel "live".

The Spinning Beach Ball

If an app exceeds its quantum repeatedly without yielding or responding, macOS flags it as unresponsive, giving you the infamous spinning beach ball.

3

Priority Scheduling

3.1 The VIP System

What if some customers are more important? The "Importance First" Algorithm allocates the CPU to the highest priority process. Generally, a lower number means a higher priority.

3.2 Preemptive vs Non-Preemptive

  • Non-Preemptive: Once the CPU is allocated, it runs to completion. Priority is only checked when the current process finishes (like a VIP waiting for the current customer to finish their order).
  • Preemptive: A higher priority process can interrupt the current process immediately (like a VIP cutting the line).

3.3 Starvation and Aging

The Starvation Problem

If high-priority processes keep arriving, low-priority processes will starve and never get executed.

The Solution: AGING. Gradually increase the priority of waiting processes over time. Eventually, even the lowest priority process becomes the highest priority and gets to run. Once it runs, its priority is reset.

4

Multi-Level Queue (MLQ)

4.1 The Separate Lines Algorithm

Processes are divided into separate queues based on their type (System, Interactive, Batch). Each queue has its own scheduling algorithm, and there is a fixed priority between the queues. A process stays in its assigned queue forever.

4.2 The Airport Analogy

Airport Security Lines
  • First Class Line (Queue 1): Shortest wait time. Always served first.
  • Business Class Line (Queue 2): Medium wait time. Only served if First Class is empty.
  • Economy Line (Queue 3): Longest wait time. Only served if Q1 and Q2 are empty.

In basic MLQ, you cannot switch lines. This leads to starvation for Economy if First Class keeps filling up!

5

Multi-Level Feedback Queue (MLFQ)

5.1 The Adaptive Learning Algorithm

MLFQ solves the starvation problem of MLQ by allowing processes to MOVE between queues. The system learns process behavior dynamically based on CPU usage and waiting time.

5.2 Promotion and Demotion

  • Demotion: If a process uses its FULL time quantum (it's a CPU hog), it gets demoted to the next lower queue.
  • Stays: If a process finishes or blocks for I/O before the quantum expires, it stays in the same queue (or gets promoted).
  • Promotion (Aging): After a certain period of waiting, all processes are moved back to the highest priority queue to prevent starvation.
6

macOS & MLFQ

6.1 Real-World Priority Hierarchy

macOS uses a sophisticated MLFQ with 128 priority levels. Here is the general hierarchy from highest to lowest:

  1. Real-Time Threads (Audio, Video): Absolute highest priority (outside standard MLFQ).
  2. User Interactive (Safari, Finder): Direct user actions like clicks and typing. Gets High priority.
  3. User Initiated: Opening files, searching.
  4. Default: General tasks, network requests.
  5. Utility: Background work, downloads.
  6. Background (Time Machine, Indexing): Maintenance tasks. Lowest priority. Can be delayed without the user noticing.
Example Scenario

If you're watching a video (Interactive - P1) while a backup runs (Background - P5), the video preempts the backup if CPU is needed. The backup only continues when the CPU is idle. This extends battery life while maintaining perfect responsiveness.
